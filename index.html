How the tracker works

Overview

This tracker is built for DITM (Deep In The Money) long call options — high-delta calls (typically 70–90 delta) with 14–21 DTE at entry. The goal is to use calls as leveraged stock substitutes, where the option moves nearly dollar-for-dollar with the underlying while risking only the premium paid.

Each trade is tracked as a campaign: a single position that may consist of multiple legs if the contract is rolled forward or up during its life. The engine evaluates exit conditions on every 90-second tick during market hours.

Position sizing

Contracts are sized so that a 1× ATR(14) move against you equals your configured risk per trade (default 5% of account). Formula:

contracts = floor(risk_budget / (delta × ATR × 100))

This means larger-ATR names get fewer contracts, and you always risk roughly the same dollar amount per trade regardless of ticker.

Exit rules — priority order

On each tick, the engine checks rules in this order and fires the first one that applies. Only one rule fires per tick.

💰 Take profit
Close when option gains the configured % from leg entry premium. Disabled when set to 0.
Any time
2
📐 Trailing stop
Tracks the option's peak price. Once profitable, closes if premium falls back the configured % from its high-water mark. Only triggers when position is in profit. Resets on each roll.
Any time (profit only)
3
📉 Regime exit
SPY 10 EMA crosses below 20 EMA — broad market turns bearish. All open positions close.
Last hour only
4
🛑 Emergency stop
Stock drops below entry − (emergency mult) × ATR. Hard floor for catastrophic moves.
Any time
5
⛔ Premium stop-loss %
Option loses the configured % of leg entry premium. Caps max loss per leg independent of stock movement. Disabled when set to 0.
Any time
6
🔴 ATR stop
Stock drops below entry − (stop mult) × ATR. Softer than emergency — catches slow deterioration.
Last hour only
7
⏱️ Time roll
When DTE ≤ trigger: find a liquid contract with higher DTE and similar delta, roll out. If no liquid contract found, close instead of rolling.
Last hour only
8
🎯 Winner roll-up
Stock reaches entry + N× ATR (for the Nth step). Roll the strike up within the configured delta band to re-center and capture more intrinsic value.

Last-hour restriction toggle: When disabled in Settings, rules 6–8 (ATR stop, time roll, winner roll-up) fire all day rather than only 3–4 PM ET. Useful if you want more active intraday management.

Regime exit toggle: When disabled, rule 3 (SPY EMA cross) is skipped entirely. Useful if you want to ignore broad market signals and let stock-level rules govern exits.

Exit strategies explained

  • Standard — Pure ATR-based stops with time roll and winner roll-up. No premium % rules. Best for mechanical DITM trading where you want the stock price to drive decisions.
  • Aggressive — Tighter stops (0.75× ATR), earlier time roll (DTE ≤ 10), take profit at +100% premium, stop-loss at −40% premium. Higher churn, smaller losses, faster exits.
  • Passive — Wider stops (1.5× ATR), late time roll (DTE ≤ 5), stop-loss at −50% premium. Fewer trades triggered, gives positions more room to breathe.
  • Trailing — Standard ATR stops plus a 25% trailing stop on premium. Lets winners run as long as the option is above its peak minus 25%. Good for trending names.
  • Custom — Set every parameter manually. Use when you want to combine specific take profit targets with ATR stops or tune stops per your risk tolerance.

Rolling mechanics

A roll closes the current leg and opens a new one in the same campaign. The P&L of the closed leg is realized and recorded; the new leg continues tracking from its own entry mark. This means campaign P&L is the sum of all leg P&Ls.

Time roll targets the nearest expiration with: delta ≥ configured min, DTE ≥ configured min, and good liquidity (OI ≥ 500, spread ≤ 10%). If nothing qualifies, the position closes outright.

Winner roll-up targets the nearest expiration with: delta in the configured band (default 65–85), DTE beyond current, and good liquidity. Rolls happen at each +1 ATR step from entry stock price.

Scorecard metrics

  • Win rate — % of closed campaigns with positive net P&L.
  • Profit factor — Gross wins ÷ gross losses. Above 1.0 = profitable overall. 2.0+ is strong.
  • Expectancy (R) — Average P&L per trade in risk units. 0.30R+ is healthy for a systematic strategy.
  • Payoff ratio — Avg win ÷ avg loss. If below 1.0 you need a high win rate to be profitable.
  • Max drawdown — Worst peak-to-trough in cumulative closed P&L. Tells you how deep the pain got.
  • Avg rolls — Average legs per campaign minus 1. Higher means more active management.

Data providers

  • Tradier (recommended) — Real-time quotes, option chains with greeks, historical OHLCV. Requires a production API token. Use sandbox for testing.
  • FMP (Financial Modeling Prep) — Daily OHLCV and stock quotes. Good for ATR and equity price; does not support option chains.
  • Alpaca — Daily bars and option snapshots. Requires separate key/secret. Option greeks quality varies.

You can mix providers — e.g., Tradier for options greeks + FMP for equity price history. Configure in Settings → Providers.

GitHub sync

The tracker stores all data in your browser's localStorage. To persist data across devices or browsers, connect a GitHub repo in Settings. The tracker will push positions.json and history.json to your repo after each change, and you can pull from another device.

API keys and your GitHub PAT are never committed — only data files are synced. Each browser stores its own keys.
